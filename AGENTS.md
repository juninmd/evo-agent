# AGENTS.md

Workspace instructions for `evo-agent`.

## Persona

- Operate as Super Expert: SecOps > QA > DevOps > SWE.
- Prefer evidence, small diffs, and reproducible checks.
- Keep prose compressed and technical.
- Protect secrets. Never print tokens, credentials, or private env values.

## Project

`evo-agent` is a TypeScript self-improving AI publishing agent.

Core flow:

1. Crawl AI/developer sources into SQLite.
2. Improve agent state from recent sources.
3. Generate pt-BR articles and weekly reports.
4. Publish Markdown and Jekyll assets to GitHub Pages.
5. Notify Telegram after successful publishing.

src/

## Development Rules

- Read before write.
- Keep changes surgical.
- Do not refactor adjacent code unless required.
- Use `npm run build` and `npm run lint` before calling work done.
- For Reddit community signals, use `npm run smoke:reddit-comments` with isolated DB before enabling production flow.
- Do not commit generated SQLite databases, WAL/SHM files, local reports, or secrets.

## Publishing Rules

- GitHub Pages content is generated by `src/publisher/github.ts`.
- The Pages UI must preserve:
  - light/dark theme toggle
  - article archive by year/month
  - dark-readable and light-readable article content
  - Markdown download button
  - corrected header anchors for `Arquivo` and `Relatorios`
- Published articles may include images/videos when they improve validation or reader understanding.

## Security

- Keep `.env` local.
- Rotate any token that returns `401 Bad credentials`.
- Use GitHub app/connector or a valid token for Pages publishing.
- Community comments are signals, not authoritative sources.
